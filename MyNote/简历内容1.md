# 对应简历内容：

- **全链路可观测能力设计**：针对 LLM 黑盒的不确定性，在业务关键节点采集输入与输出等核心指标的 Trace 事件，落库可回溯，支撑线上排查逐步回放，为每一次用户对话提供全链路可观测能力支撑；

### 1. 背景阐述

Agent 系统与传统的后端系统有所不同

因为引入了大模型，变成了一个不确定性较大的黑盒

即 Agent 系统的每一次处理的结果可能都会不一样

因此每一次 Agent 系统的接口请求，我们必须能够知道这一次请求每一步做了什么

调用了哪些 Agent、哪些模型、模型的输入和输出是什么，token 用量多少

在 Agent 构建的整个业务系统之下 每一步的流程、每个业务节点的输入和输出又是什么

所以要把这个运行中的类似黑盒的 Agent 系统的细节始终掌握

**可观测性与监控能力的建设是第一步**

做好了可观测性与监控能力之后，才有基础去谈后续的评估与优化迭代

### 2. 实现方案

针对当前这个饮食推荐系统，我在业务的关键节点和 Agent 调用时都会记录 Trace 到内存的 ThreadLocal 之中

例如 意图识别和槽位信息提取时记录下 意图识别的结果和提取到的槽位信息

是否需要澄清、检索餐食，餐食排序、最终应答等各个关键环节 都记录相对应的信息

最后在一次请求结束返回时，将其写入数据库

因此这一次请求的每一步流程，通过这个 Trace 就可以完全回溯

通过这个全链路监控，我能够知道系统请求 的耗时、大模型的调用次数与耗时、Token的成本、意图识别的结果、是否需要澄清的决策、检索和排序的结果、最终应答的结果。这些都是我监控的维度

### 3. 复盘总结

最后总结一下，可观测性的能力建设，将 "黑盒" 特性的Agent 系统的透明度提升了

且 Trace 数据是后续对系统进行持续优化迭代的数据基础，是支撑整个系统改进的基石

## **Trace 的实现 && 一次 Trace 的解读**

### 1. Trace 的实现

在整个系统开始处理使用 openTrace 在当前线程建立 Trace 上下文

整轮处理过程中通过 recordEvent/recordError/callAgent 把事件追加到内存列表

try-with-resources 结束时 TraceScope.close() 将汇总信息 + 全部 events 序列化为 trace_json

一次性 INSERT 到 diet_request_trace 表，整轮结束才 INSERT 一次，减少 DB 压力

```
// 开启 Trace 上下文；try-with-resources 结束时 TraceScope#close 会将整轮事件写入 agent_traces 表
try (AgentTraceService.TraceScope ignored = agentTraceService.openTrace(traceId, initialState.sessionId(), userId)) {
    try {
        // 记录请求开始时间（纳秒），用于最后计算整轮耗时
        long startedAt = System.nanoTime();
        // Trace 事件：REQUEST_RECEIVED | 阶段 HTTP | 输入=ChatRequest | 输出=初始 SessionState
        agentTraceService.recordEvent("REQUEST_RECEIVED", "HTTP", request, initialState);

        // 获取或创建该 sessionId 对应的锁对象，保证同一 session 并发请求串行执行
        Object lock = sessionLocks.computeIfAbsent(initialState.sessionId(), key -> new Object());
        synchronized (lock) {
            // 在锁内执行完整状态机，处理本轮用户输入
            ChatResponse response = handleTurn(userId, request, traceId, initialState);
            // Trace 事件：REQUEST_FINISHED | 阶段 HTTP | 输入=ChatRequest | 输出=ChatResponse | 耗时 ms
            agentTraceService.recordEvent("REQUEST_FINISHED", "HTTP", request, response, elapsedMs(startedAt));
            // 将最终响应返回给 Controller
            return response;
        }
    } catch (RuntimeException error) {
        // Trace 事件：REQUEST_FAILED | 阶段 HTTP | 输入=ChatRequest | 记录异常并将 Trace 标记 FAILED
        agentTraceService.recordError("REQUEST_FAILED", "HTTP", request, error);
        // 继续向上抛出，由全局异常处理器返回错误响应
        throw error;
    }
}
```
